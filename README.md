# Fx
Clean architecture

---

## 原则分为多个层次：

> Domain层 - 包含业务逻辑实体和接口。

> Application层 - 包含业务逻辑实现（服务、DTO等）。

> Infrastructure层 - 处理外部依赖（例如数据库、第三方服务等）。

> API层 - 处理Web API接口请求。

---

## Git 贡献提交规范

>feat 增加新功能

>fix 修复问题/BUG

>style 代码风格相关无影响运行结果的

>perf 优化/性能提升

>refactor 重构

>revert 撤销修改

>test 测试相关

>docs 文档/注释

>chore 依赖更新/脚手架配置修改等

>ci 持续集成

>types 类型定义文件更改

>wip 开发中
